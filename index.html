Quality Assurance Testing on SPACE & BEYOND

Introduction

Testing is a very important part of software development. It ensures that applications function correctly and meet user expectations. Thanks to testing, we can identify errors and improve the quality of the final product. With this in mind, I created this website to showcase the work I did in the QA Testing course at Coderhouse. Creating this page helps me review what I learned in class about testing and continue practicing my web development skills. I believe there is no better way to apply what I've learned than through this project.

Performance Analysis

In the current context, where user experience is fundamental to the success of any website, conducting performance tests is essential. Below is a comprehensive analysis of the performance of the Space & Beyond page, using the Lighthouse tool to evaluate both the mobile and desktop versions. Through key metrics, areas for improvement are identified, and achievements in site optimization are highlighted.

Desktop performance analysis
Mobile performance analysis
Desktop performance analysis
Overall Performance
The desktop performance of Space & Beyond is 95, indicating a green status (90 to 100). This suggests that the page is well-optimized and provides an excellent user experience.
Key Metrics
  1. First Contentful Paint (FCP): 1.0 s
    • Analysis: An FCP of 1.0 seconds is good, indicating that the first visual content is displayed effectively.
  2. Largest Contentful Paint (LCP): 1.2 s
    • Analysis: An LCP of 1.2 seconds is satisfactory, meaning that the largest visible content loads quickly.
  3. Total Blocking Time (TBT): 30 ms
    • Analysis: A TBT of 30 ms is low, indicating that the page is quite responsive.
  4. Cumulative Layout Shift (CLS): 0
    • Analysis: A CLS of 0 means there is no unexpected movement of elements during loading.
  5. Speed Index: 1.4 s
    • Analysis: With a Speed Index of 1.4 seconds, the page displays its content in a reasonable time.
Conclusion
The desktop performance is excellent, with metrics that meet and exceed suggested levels to provide an ideal experience. It is recommended to maintain current optimization practices and conduct regular tests to ensure performance does not degrade.
Mobile performance analysis
Overall Performance
The mobile performance of Space & Beyond is rated at 71, indicating an orange status (50 to 89). This suggests that while the page is functional, there are areas that could be improved to optimize the user experience.
Key Metrics
  1. First Contentful Paint (FCP): 3.4 s
    • Analysis: An FCP of 3.4 seconds indicates that the first visual content takes longer to display, which could affect user experience.
  2. Largest Contentful Paint (LCP): 4.8 s
    • Analysis: An LCP of 4.8 seconds is considered slow, meaning that the largest visible content takes a significant amount of time to load.
  3. Total Blocking Time (TBT): 220 ms
    • Analysis: A TBT of 220 ms indicates some responsiveness issues, as tasks are blocking the main thread for a noticeable period.
  4. Cumulative Layout Shift (CLS): 0.045
    • Analysis: A CLS of 0.045 suggests minor layout shifts during loading, which can lead to a slightly unstable user experience.
  5. Speed Index: 3.4 s
    • Analysis: With a Speed Index of 3.4 seconds, the page shows its content at a slower rate, which may impact user engagement.
Conclusion
Although the mobile performance presents critical areas to address, metrics such as TBT and CLS are acceptable. It is recommended to work on reducing both FCP and LCP to enhance the overall perception of performance.

The performance analysis of Space & Beyond shows a clear difference between the mobile and desktop versions of the website. The desktop version has a very high performance, with metrics indicating that the experience is fast and smooth. In contrast, the mobile version presents areas that need improvement for users to have a better experience. To achieve a good balance between both platforms, it is important to make specific enhancements to the mobile version, focusing on reducing load times and ensuring that interactions are faster and more efficient. This report serves as a solid foundation for future improvements, ensuring that Space & Beyond can provide an excellent experience to all its users, regardless of the device they use.

In the following work, a research study is presented on the APIs used on the Space & Beyond website. The objective of this activity is to analyze the requests made to different endpoints and evaluate their status. Through the use of the element inspector, several API requests have been identified, some of which present errors that could affect the functionality of the site. This analysis will allow us to better understand how web applications interact with their resources and detect possible areas for improvement.
