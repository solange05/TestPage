SPACE & BEYOND

Quality Assurance Testing on SPACE & BEYOND

Introduction

Testing is a very important part of software development. It ensures that applications function correctly and meet user expectations. Thanks to testing, we can identify errors and improve the quality of the final product. With this in mind, I created this website to showcase the work I did in the QA Testing course at Coderhouse. Creating this page helps me review what I learned in class about testing and continue practicing my web development skills. I believe there is no better way to apply what I've learned than through this project.

Bug Report

It is essential to document each software error with details such as severity, expected and actual results, evidence, reproduction steps, bug type, and the browser used. Similarly, test cases should include a clear name, objective, steps to follow, preconditions, expected results, and current status. This precise documentation is key to identifying and resolving errors, ensuring software quality.

Test Report

In this report, a total of 10 errors were identified. Of these, 7 high-priority errors were corrected, representing 70% of the total. However, the remaining 3 errors, classified as low priority, could not be resolved due to time constraints, which accounts for 30% of the total errors. This analysis shows that the team did a good job addressing the most critical issues. However, it is also necessary to resolve the low-priority errors in the future to improve product quality. Time management and task organization are key to ensuring a good user experience.
